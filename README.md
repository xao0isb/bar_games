# Flappy Bird — управление по QR-коду

Игра **Flappy Bird** выводится на большом экране (главная странице). Игрок сканирует
QR-код телефоном, открывает страницу с одной большой кнопкой и управляет птичкой:
каждое нажатие кнопки на телефоне заставляет птицу на большом экране прыгать.

```
┌─────────────┐     сканирует QR      ┌───────────────┐
│ Большой     │  ───────────────────▶ │  Телефон      │
│ экран  🐦   │                       │  [ ПРЫЖОК ]   │
│ (главная)   │ ◀── "flap" по WS ──── │  (пульт)      │
└─────────────┘                       └───────────────┘
```

Стек: **Python + FastAPI**, WebSocket для реального времени, игра на HTML5 Canvas.

## Запуск

```bash
./run.sh
```

Скрипт создаст виртуальное окружение, установит зависимости и запустит сервер на
порту **8000**. Либо вручную:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000
```

## Как играть

1. **Важно:** откройте главную страницу по **IP-адресу машины в локальной сети**, а не
   через `localhost` — иначе QR-код будет указывать на `localhost`, и телефон не сможет
   подключиться.

   Узнать IP: `hostname -I` (Linux) или `ipconfig getifaddr en0` (macOS). Например:

   ```
   http://192.168.1.50:8000
   ```

2. Телефон и компьютер должны быть в **одной Wi-Fi сети**. При необходимости откройте
   порт 8000 в файрволе.
3. На экране появится QR-код. Отсканируйте его телефоном — откроется страница-пульт.
4. Как только телефон подключится, на экране появится «Приготовьтесь!». Нажмите кнопку
   на телефоне — птичка взлетит и игра начнётся.
5. После проигрыша нажмите кнопку ещё раз, чтобы сыграть заново.

## Как это устроено

| Файл | Назначение |
|------|-----------|
| `main.py` | FastAPI-приложение: страницы, генерация QR (SVG), WebSocket-релей |
| `templates/index.html` | Главный экран с игрой |
| `templates/controller.html` | Страница-пульт для телефона |
| `static/host.js` | Логика и отрисовка Flappy Bird (Canvas) |
| `static/controller.js` | Кнопка + отправка событий по WebSocket |
| `static/style.css` | Стили обеих страниц |

Сервер сам игру не считает — он только пересылает события. Физика и отрисовка птички
идут в браузере на большом экране (60 fps), поэтому картинка плавная, а по сети летают
только короткие сообщения `flap` / `state`.

### Сессии

Каждый экран получает свой `session_id` (в адресе `/?s=...`), поэтому одновременно можно
запустить несколько независимых игр. Обновление главной страницы сохраняет тот же
QR-код. WebSocket-адреса:

* `ws://<host>/ws/<session_id>/host` — большой экран;
* `ws://<host>/ws/<session_id>/controller` — телефон.

### Управление на десктопе (для теста)

На странице-пульте работает клавиша **Пробел** / **↑**, а на самом экране игру можно
проверить, открыв пульт во второй вкладке браузера.
